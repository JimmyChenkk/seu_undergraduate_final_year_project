\chapter{基线方法复现与 Benchmark 结果分析}

\section{实验目的与设置}
本章的目标是基于统一的 FCN backbone 和统一训练协议，对当前已经实现的典型领域自适应方法进行 benchmark 对比，明确不同方法在 TEP 多工况迁移场景中的表现差异，并为改进方法设计提供问题依据。考虑到全量 72 组迁移场景的实验量较大，正文先选取六组代表性单源到单目标场景进行集中展示，并结合已完成的部分多源场景做补充讨论。六组代表性单源场景分别为：\texttt{mode1$\rightarrow$mode4}、\texttt{mode4$\rightarrow$mode1}、\texttt{mode2$\rightarrow$mode5}、\texttt{mode5$\rightarrow$mode2}、\texttt{mode3$\rightarrow$mode6} 和 \texttt{mode6$\rightarrow$mode3}。

在实验协议上，各方法均采用相同的一维 FCN 特征提取网络、相同的数据划分方式与相同的训练轮数设置。目标域训练阶段不使用真实标签，仅在测试阶段统计目标域准确率与平衡准确率。与此同时，实验流程还会记录源域训练准确率、源域评估准确率以及结果汇总文件，用于分析方法是否存在“源域性能下降但目标域收益有限”的对齐失衡现象。

\section{FCN 骨干网络与 Source Only 基线}
本文采用一维全卷积网络作为统一特征提取 backbone。该网络由三层一维卷积、归一化和非线性映射构成，并通过时间维平均池化得到紧凑特征表示。相比直接引入复杂时序结构，FCN 在当前 TEP benchmark 中具有实现简单、训练稳定和便于跨方法公平比较等优点，因此被用作本研究的统一主干网络。

Source Only 方法仅依赖源域监督分类损失进行训练，不显式处理源域与目标域之间的分布偏移，因此可视为所有领域自适应方法的基础对照组。若某一领域自适应方法在目标域上不能显著优于 Source Only，则说明其对齐策略尚未真正缓解跨工况迁移带来的分布差异问题。

\section{基线方法复现说明}
\subsection{分布对齐基线}
CORAL 通过约束源域与目标域特征协方差矩阵的一致性来减小二阶统计差异~\cite{sun2016deep}；DAN 则使用多核最大均值差异来约束特征分布，在核空间中实现更细致的边缘分布对齐~\cite{long2015dan}。这两类方法的共同特点是训练目标较为直接，通常能在不引入额外判别器的情况下稳定收敛，因此适合作为 benchmark 中的基础分布对齐参照。

\subsection{对抗式基线}
DANN 在共享特征提取网络之后加入域判别器，并利用梯度反转层实现特征表示与域分类之间的对抗学习，从而逼近域不变特征~\cite{ganin2016dann}。CDAN 则进一步将类别预测结果纳入条件化对齐过程，使对抗过程能够感知类别结构信息，以缓解简单全局对齐造成的类别混叠问题~\cite{long2018conditional}。在本工作中，两类方法均已基于当前 benchmark 主线完成统一实现与调度策略补充。

\subsection{最优传输基线}
DeepJDOT 通过联合建模特征空间距离与类别预测差异，构建源域和目标域样本之间的运输代价，在深度特征学习过程中同时优化分类与跨域匹配~\cite{damodaran2018deepjdot}。该方法能够提供样本级的跨域关系约束，但也更容易受到批大小、代价尺度和训练稳定性的影响。

进一步比较每个具体场景中的相对变化可以发现，CORAL、DAN、DANN 和 DeepJDOT 在已完成的六组代表性单源场景中均优于 Source Only，说明引入域对齐机制对于 TEP 跨工况故障诊断是必要且有效的。CDAN 在四组场景中优于 Source Only，但在 \texttt{mode4$\rightarrow$mode1} 和 \texttt{mode6$\rightarrow$mode3} 两组场景中的优势并不稳定，表明条件对齐策略在当前实现下仍对场景分布和训练配置较为敏感。

\subsection{分场景观察}
从已完成场景的最好结果来看，\texttt{mode3$\rightarrow$mode6}、\texttt{mode2$\rightarrow$mode5} 与 \texttt{mode5$\rightarrow$mode2} 三组场景中，DAN 与 DANN 表现相对更强，目标域准确率已达到约 0.80 以上；在 \texttt{mode1$\rightarrow$mode4} 这一较难场景中，CDAN 的目标域准确率达到了 0.8126，显著高于 Source Only 的 0.3608，体现出条件化对齐在复杂类别结构迁移中仍具有潜力。另一方面，DeepJDOT 的平均收益相对有限，说明最优传输类方法在当前时序故障诊断任务上还需要进一步优化代价构造与训练稳定性。

已完成的多源到单目标场景结果也反映出一个重要现象：当源域信息更加充分时，Source Only 本身已经能够达到较高的目标域准确率，例如当前已完成的 \texttt{mode2-mode3-mode4-mode5-mode6$\rightarrow$mode1} 场景中，Source Only 已达到 0.7586，而 CORAL 进一步提升到了 0.8241。这说明多源信息融合本身是一个值得继续深入的方向，但同时也提示单纯依靠更多源域数据并不能完全替代更细粒度的类条件对齐机制。

\section{结果分析与改进动机}
\subsection{当前 benchmark 揭示的主要问题}
结合已完成实验结果，可以总结出当前 benchmark 暴露出的三个主要问题。第一，Source Only 在源域上通常能够获得较高准确率，但在目标域上的性能明显下降，说明跨工况分布偏移对故障诊断模型的影响依然显著。第二，分布对齐方法虽能整体提升目标域性能，但不同方法在不同场景上的稳定性存在明显差异。例如 CDAN 在部分场景中表现突出，但并非在所有场景中都优于 DAN 或 DANN。第三，部分方法在目标域上取得收益的同时，会出现源域分类能力下降或训练过程更敏感的情况，这说明“全局域对齐”与“类别判别能力”之间仍存在权衡。

\subsection{对创新方法设计的启示}
上述现象表明，当前问题的关键并不只是“是否进行对齐”，而是“如何在保持类别判别能力的前提下进行更稳定的对齐”。现有全局边缘分布对齐方法能够减小整体域差异，但无法显式保证跨域同类样本靠近、异类样本保持分离；而条件对齐方法虽然引入了类别信息，却仍容易受到伪标签噪声和训练不稳定性的影响。因此，改进方法需要同时兼顾以下三点：其一，保留当前 benchmark 中已经验证有效的全局域对齐机制；其二，引入更直接的类别结构约束以减少跨域类别混叠；其三，通过置信度筛选和渐进式权重调度控制目标域伪标签噪声对训练的负面影响。

\section{本章小结}
本章在统一 FCN backbone 和统一实验协议下，对 Source Only、CORAL、DAN、DANN、CDAN 与 DeepJDOT 等基线方法进行了 benchmark 比较。已完成的六组代表性单源场景结果表明，领域自适应方法整体上显著优于 Source Only，其中 DAN 与 DANN 的平均目标域表现较强，而 CDAN 在个别困难场景中展现出更高上限但稳定性仍需提升。基于这些结果，本文把创新重点进一步收敛到“如何利用类别结构信息提升跨域同类聚合能力”这一问题上，下一章将据此给出一种更具体的原型引导类条件对齐方案。
