\chapter{基线方法与实验结果分析}

\section{基线方法说明}
本章围绕六种基线方法展开比较，包括 Source Only、CORAL、DAN、DANN、CDAN 和 DeepJDOT。所有方法共享第三章介绍的 FCN 骨干网络，因此方法差异主要体现在损失函数与对齐机制上。

\subsection{Source Only}
Source Only 仅依赖源域监督分类损失训练模型，不显式处理源域与目标域之间的分布偏移，其优化目标可写为
\begin{equation}
\mathcal{L}_{SO}=\mathcal{L}_{cls}^s.
\end{equation}
该方法可视为所有领域自适应方法的基础对照组。若某一领域自适应方法在目标域上不能显著优于 Source Only，则说明其对齐策略尚未真正缓解跨工况迁移带来的分布差异问题。

\subsection{分布对齐方法}
CORAL 通过约束源域与目标域特征协方差矩阵的一致性来减小二阶统计差异~\cite{sun2016deep}。设源域与目标域特征分别为 $F_s\in\mathbb{R}^{n_s\times d}$ 和 $F_t\in\mathbb{R}^{n_t\times d}$，对应均值向量与协方差矩阵为 $\mu_s,\mu_t,C_s,C_t$，则 CORAL 损失为
\begin{equation}
\mathcal{L}_{coral}=
\frac{\|C_s-C_t\|_F^2}{4d^2}
+\|\mu_s-\mu_t\|_2^2.
\end{equation}
总损失写为
\begin{equation}
\mathcal{L}_{CORAL}=\mathcal{L}_{cls}^s+\lambda_{coral}\mathcal{L}_{coral}.
\end{equation}

DAN 使用多核最大均值差异（MK-MMD）约束源域与目标域特征分布，在核空间中实现更细致的边缘分布对齐~\cite{long2015dan}。给定核函数族 $\{k_u\}_{u=1}^{U}$，其损失可表示为
\begin{equation}
\mathcal{L}_{mmd}=
\left\|
\frac{1}{n_s}\sum_{i=1}^{n_s}\varphi(f_i^s)-
\frac{1}{n_t}\sum_{j=1}^{n_t}\varphi(f_j^t)
\right\|_{\mathcal{H}}^2,
\end{equation}
实际实现中采用多个高斯核的加权和，于是总损失为
\begin{equation}
\mathcal{L}_{DAN}=\mathcal{L}_{cls}^s+\lambda_{mmd}\mathcal{L}_{mmd}.
\end{equation}
这两类方法的共同特点是训练目标较为直接，通常能在不引入额外判别器的情况下稳定收敛，因此适合作为基础分布对齐参照。

\subsection{对抗式方法}
DANN 在共享特征提取网络之后加入域判别器，并利用梯度反转层实现特征表示与域分类之间的对抗学习，从而逼近域不变特征~\cite{ganin2016dann}。设域标签 $d=1$ 表示源域，$d=0$ 表示目标域，域判别器为 $D(\cdot)$，则对抗损失可写为
\begin{equation}
\mathcal{L}_{adv}=
-\frac{1}{n_s}\sum_{i=1}^{n_s}\log D(f_i^s)
-\frac{1}{n_t}\sum_{j=1}^{n_t}\log(1-D(f_j^t)).
\end{equation}
在梯度反转层作用下，特征提取器朝着“混淆域判别器”的方向更新，因此总损失为
\begin{equation}
\mathcal{L}_{DANN}=\mathcal{L}_{cls}^s+\lambda_{adv}\mathcal{L}_{adv}.
\end{equation}

CDAN 在 DANN 基础上进一步将类别预测结果纳入条件化对齐过程，使域判别器感知类别结构信息~\cite{long2018conditional}。若记分类器输出概率为 $p(x)$，则 CDAN 对特征与类别预测进行联合映射：
\begin{equation}
g(x)=f(x)\otimes p(x),
\end{equation}
其中 $\otimes$ 表示多线性组合。域判别器不再直接作用于 $f(x)$，而是作用于 $g(x)$。当前实现中还支持随机映射版本与熵加权机制，并可附加最小类混淆损失（MCC）：
\begin{equation}
\mathcal{L}_{CDAN}=
\mathcal{L}_{cls}^s+\lambda_{cadv}\mathcal{L}_{cadv}+\lambda_{mcc}\mathcal{L}_{mcc}.
\end{equation}
这两类方法分别代表了全局对抗对齐和条件对抗对齐两种典型思路。

\subsection{最优传输方法}
DeepJDOT 通过联合建模特征空间距离与类别预测差异，构建源域和目标域样本之间的运输代价，在深度特征学习过程中同时优化分类与跨域匹配~\cite{damodaran2018deepjdot}。设 $\gamma$ 为源域与目标域样本间的运输矩阵，则其代价函数可写为
\begin{equation}
\mathcal{L}_{OT}=
\min_{\gamma\in\Pi(a,b)}
\sum_{i,j}\gamma_{ij}
\left(
\alpha\|f_i^s-f_j^t\|_2^2+
\beta\ell(y_i^s,p_j^t)
\right),
\end{equation}
其中 $\Pi(a,b)$ 表示满足边缘约束的运输计划集合，$\alpha$ 和 $\beta$ 分别控制特征距离代价与分类代价的权重。于是总损失写为
\begin{equation}
\mathcal{L}_{DeepJDOT}=\mathcal{L}_{cls}^s+\lambda_{ot}\mathcal{L}_{OT}.
\end{equation}
该方法能够提供样本级跨域关系约束，但也更容易受到批大小、代价尺度和计算开销的影响。

\section{实验范围与比较协议}
本章聚焦四个关键单源场景：mode2$\rightarrow$mode5、mode3$\rightarrow$mode6、mode5$\rightarrow$mode2 和 mode6$\rightarrow$mode3。所有结果均来自第三章给出的统一实验协议，即共享 FCN 骨干、批大小 32、学习率 $10^{-4}$、固定 Fold 1 划分以及一致的模型选择规则。除方法本身的对齐机制外，其余训练流程尽量保持一致，以便将性能差异尽量归因于适应机制本身。

在超参数层面，DAN 的 MK-MMD 权重为 0.05，DANN 的对抗权重为 0.5，CDAN 的条件对抗权重为 0.2，DeepJDOT 的 OT 权重为 1.0。该设置与论文附录中的配置表保持一致。

\subsection{分场景结果分析}
从分场景角度看，不同迁移方向之间存在明显差异，这一点对后续方法设计具有直接启示。

\paragraph{1. mode2$\rightarrow$mode5 场景}
该场景中 Source Only 的目标域准确率仅为 0.6094，说明 mode2 与 mode5 之间存在较明显的分布偏移。DAN、DANN 和 CDAN 均将目标域准确率提升到 0.81 以上，其中 DANN 达到 0.8299，为基线最佳结果。这表明在该场景中，全局对抗对齐和核分布对齐都能够有效缓解跨域偏移。

\paragraph{2. mode3$\rightarrow$mode6 场景}
Source Only 已达到 0.7064，但 DANN 仍可进一步提升至 0.8566，提升幅度为 0.1503；DAN 和 CDAN 也均超过 0.818。该结果说明 mode3 与 mode6 虽然具有一定天然可迁移性，但通过显式对齐仍能获得显著收益。

\paragraph{3. mode5$\rightarrow$mode2 场景}
这是四个关键场景中最困难的方向之一，Source Only 仅为 0.5203。DAN 将目标域准确率提升到 0.8131，是本场景中最强基线；DANN 和 CDAN 分别达到 0.7831 和 0.7478。该现象说明同一对运行模式在互逆方向上的迁移难度并不对称，最优方法也可能发生变化。

\paragraph{4. mode6$\rightarrow$mode3 场景}
该场景中 Source Only 已经达到 0.8463，意味着 mode6 与 mode3 之间的天然差距相对较小。此时如果对齐策略过强，反而可能破坏已存在的有效判别结构。CORAL、DAN、CDAN 和 DeepJDOT 在该场景上均未优于 Source Only，仅 DANN 略微提升至 0.8618。这说明“域对齐越强越好”并不成立，对齐压力的控制同样重要。

\subsection{主要现象归纳}
结合表~\ref{tab:baseline-single-source} 和上述分场景分析，可以总结出以下三点。

\begin{enumerate}
\item \textbf{跨工况分布偏移是目标域性能下降的直接原因。} Source Only 在源域上通常能够获得较高准确率，但在目标域上的性能下降十分明显，尤其是在 mode5$\rightarrow$mode2 这一困难方向上表现突出。
\item \textbf{全局对齐有效，但仍然存在边界。} DAN 和 DANN 表明，只要能够有效缩小源域与目标域整体分布差异，目标域性能就能获得显著提升；然而在 mode6$\rightarrow$mode3 这类本身较易迁移的场景中，过强对齐仍可能带来负迁移。
\item \textbf{类别结构利用仍不充分。} CDAN 在理论上引入了条件信息，但在当前场景中的平均表现未能持续超越 DANN 与 DAN，这表明仅把类别预测接入域判别器仍不足以稳定解决多类别时序任务中的伪标签噪声与类混叠问题。
\end{enumerate}

\section{对改进方法设计的启示}
基线结果表明，后续方法设计不能只回答“是否对齐”，还需要回答“如何在对齐的同时维持类别结构，并避免目标域噪声监督失控”。具体而言，至少需要满足以下三个要求：

\begin{enumerate}
\item 保留当前已经验证有效的强基线对齐能力，避免完全脱离已有经验重新设计；
\item 引入更直接的类条件结构约束，使跨域同类样本更容易聚合、异类样本更容易分离；
\item 对目标域伪标签进行可靠性控制，使无标签监督在训练早期不过度扰动分类边界，在训练后期又能逐步发挥作用。
\end{enumerate}

第五章提出的 RCTA 正是围绕这三点展开：以强基线对齐分支为底座，通过教师--学生时序一致性、可靠性门控和双原型记忆，把全局域对齐与类条件结构学习结合起来。

\section{本章小结}
本章在统一 FCN 骨干网络和统一实验协议下，对 Source Only、CORAL、DAN、DANN、CDAN 与 DeepJDOT 等基线方法进行了比较。结果表明，领域自适应方法整体上显著优于 Source Only，其中 DANN 的平均目标域表现最强，DAN 则展现出较好的跨场景稳定性。与此同时，基线实验也揭示出场景难度不对称、全局对齐不足以及类条件结构利用不稳定等问题，这为下一章提出 RCTA 提供了明确的问题背景。
