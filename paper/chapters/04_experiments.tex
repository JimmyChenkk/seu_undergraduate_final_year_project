\chapter{基线方法与实验结果分析}

\section{基线方法说明}
本章围绕六种基线方法展开比较，包括 Source Only、CORAL、DAN、DANN、CDAN 和 DeepJDOT。所有方法共享第三章中介绍的 FCN 骨干网络，因此方法差异主要体现在损失函数与对齐机制上。

\subsection{Source Only}
Source Only 仅依赖源域监督分类损失训练模型，不显式处理源域与目标域之间的分布偏移，其优化目标可写为
\begin{equation}
\mathcal{L}_{SO}=\mathcal{L}_{cls}^s.
\end{equation}
该方法可视为所有领域自适应方法的基础对照组。若某一领域自适应方法在目标域上不能显著优于 Source Only，则说明其对齐策略尚未真正缓解跨工况迁移带来的分布差异问题。

\subsection{分布对齐方法}
CORAL 通过约束源域与目标域特征协方差矩阵的一致性来减小二阶统计差异~\cite{sun2016deep}。设源域与目标域特征分别为 $F_s\in\mathbb{R}^{n_s\times d}$ 和 $F_t\in\mathbb{R}^{n_t\times d}$，对应均值向量与协方差矩阵为 $\mu_s,\mu_t,C_s,C_t$，则本文实现中的 CORAL 损失为
\begin{equation}
\mathcal{L}_{coral}=
\frac{\|C_s-C_t\|_F^2}{4d^2}
+\,
\|\mu_s-\mu_t\|_2^2.
\end{equation}
总损失写为
\begin{equation}
\mathcal{L}_{CORAL}=\mathcal{L}_{cls}^s+\lambda_{coral}\mathcal{L}_{coral}.
\end{equation}

DAN 使用多核最大均值差异（MK-MMD）约束源域与目标域特征分布，在核空间中实现更细致的边缘分布对齐~\cite{long2015dan}。给定核函数族 $\{k_u\}_{u=1}^{U}$，其损失可表示为
\begin{equation}
\mathcal{L}_{mmd}=
\left\|
\frac{1}{n_s}\sum_{i=1}^{n_s}\varphi(f_i^s)-
\frac{1}{n_t}\sum_{j=1}^{n_t}\varphi(f_j^t)
\right\|_{\mathcal{H}}^2,
\end{equation}
实际实现中采用多个高斯核的加权和。于是总损失为
\begin{equation}
\mathcal{L}_{DAN}=\mathcal{L}_{cls}^s+\lambda_{mmd}\mathcal{L}_{mmd}.
\end{equation}
这两类方法的共同特点是训练目标较为直接，通常能在不引入额外判别器的情况下稳定收敛，因此适合作为基础分布对齐参照。

\subsection{对抗式方法}
DANN 在共享特征提取网络之后加入域判别器，并利用梯度反转层实现特征表示与域分类之间的对抗学习，从而逼近域不变特征~\cite{ganin2016dann}。设域标签 $d=1$ 表示源域，$d=0$ 表示目标域，域判别器为 $D(\cdot)$，则对抗损失可写为
\begin{equation}
\mathcal{L}_{adv}=
-\frac{1}{n_s}\sum_{i=1}^{n_s}\log D(f_i^s)
-\frac{1}{n_t}\sum_{j=1}^{n_t}\log(1-D(f_j^t)).
\end{equation}
在梯度反转层作用下，特征提取器朝着“混淆域判别器”的方向更新，因此总损失为
\begin{equation}
\mathcal{L}_{DANN}=\mathcal{L}_{cls}^s+\lambda_{adv}\mathcal{L}_{adv}.
\end{equation}

CDAN 在 DANN 基础上进一步将类别预测结果纳入条件化对齐过程，使域判别器感知类别结构信息~\cite{long2018conditional}。若记分类器输出概率为 $p(x)$，则 CDAN 对特征与类别预测进行联合映射：
\begin{equation}
g(x)=f(x)\otimes p(x),
\end{equation}
其中 $\otimes$ 表示多线性组合。域判别器不再直接作用于 $f(x)$，而是作用于 $g(x)$。当前实现中还支持随机映射版本与熵加权机制，并可附加最小类混淆损失（MCC）：
\begin{equation}
\mathcal{L}_{CDAN}=
\mathcal{L}_{cls}^s+\lambda_{cadv}\mathcal{L}_{cadv}+\lambda_{mcc}\mathcal{L}_{mcc}.
\end{equation}
这两类方法分别代表了全局对抗对齐和条件对抗对齐两种典型思路。

\subsection{最优传输方法}
DeepJDOT 通过联合建模特征空间距离与类别预测差异，构建源域和目标域样本之间的运输代价，在深度特征学习过程中同时优化分类与跨域匹配~\cite{damodaran2018deepjdot}。设 $\gamma$ 为源域与目标域样本间的运输矩阵，则其代价函数可写为
\begin{equation}
\mathcal{L}_{OT}=
\min_{\gamma\in\Pi(a,b)}
\sum_{i,j}\gamma_{ij}
\left(
\alpha\|f_i^s-f_j^t\|_2^2+
\beta\ell(y_i^s,p_j^t)
\right),
\end{equation}
其中 $\Pi(a,b)$ 表示满足边缘约束的运输计划集合，$\alpha$ 和 $\beta$ 分别控制特征距离代价与分类代价的权重。于是总损失写为
\begin{equation}
\mathcal{L}_{DeepJDOT}=\mathcal{L}_{cls}^s+\lambda_{ot}\mathcal{L}_{OT}.
\end{equation}
该方法能够提供样本级跨域关系约束，但也更容易受到批大小、代价尺度和计算开销的影响。

\section{实验批次说明与当前统计范围}
本章结果主要来自当前正在执行的 \texttt{rcta\_70\_run\_aggressive} 批次。该批次的总计划规模为 10 个场景与 7 种方法组合形成的 70 次运行，其中包括 4 个单源场景和 6 个五源到单目标场景。写作本章时，4 个单源场景共 28 次运行已经全部完成，多源部分仍在继续，因此本章先对已经完成且可直接核验的单源结果进行系统讨论。

在超参数层面，6 种基线方法均采用统一 FCN 骨干、批大小 32、学习率 $10^{-4}$ 和基础 70 轮训练预算；其中 DAN 的 MK-MMD 权重为 0.05，DANN 的对抗权重为 0.5，CDAN 的条件对抗权重为 0.2，DeepJDOT 的 OT 权重为 1.0。除方法本身的对齐机制外，其余训练流程尽量保持一致。

\subsection{分场景结果分析}
从分场景角度看，不同迁移方向之间存在明显差异。

\paragraph{1. mode2$\rightarrow$mode5 场景}
该场景中 Source Only 的目标域准确率仅为 0.6094，说明 mode2 与 mode5 之间存在较明显的分布偏移。DAN、DANN 和 CDAN 均将目标域准确率提升到 0.81 以上，其中 DANN 达到 0.8299，为基线最佳结果。这表明在该场景中，对抗式与 MMD 式对齐都能有效缓解跨域偏移。

\paragraph{2. mode3$\rightarrow$mode6 场景}
Source Only 已达到 0.7064，但 DANN 进一步提升到 0.8566，提升幅度为 0.1503；DAN 和 CDAN 也均超过 0.818。该结果说明 mode3 与 mode6 虽然具有一定可迁移性，但通过显式对齐仍能获得相当明显的性能收益。

\paragraph{3. mode5$\rightarrow$mode2 场景}
这是当前 4 个场景中最困难的方向之一，Source Only 仅为 0.5203。DAN 将目标域准确率提升到 0.8131，是本场景中最强基线；DANN 和 CDAN 分别达到 0.7831 和 0.7478。该现象说明不同方向上的迁移难度并不对称，同一对 mode 在互逆方向上的最优方法也可能不同。

\paragraph{4. mode6$\rightarrow$mode3 场景}
该场景中 Source Only 已经达到 0.8463，意味着 mode6 与 mode3 之间的天然差距相对较小。此时，若对齐策略过强，反而可能破坏已存在的有效判别结构。CORAL、DAN、CDAN 和 DeepJDOT 在该场景上均未优于 Source Only，仅 DANN 略微提升至 0.8618。这说明“域对齐越强越好”并不成立，合理控制对齐压力同样重要。

\subsection{源域表现与目标域收益之间的关系}
在多数场景下，Source Only 的源域评估准确率均在 0.87 左右甚至更高，但目标域性能仍明显下降。例如，mode5$\rightarrow$mode2 场景中 Source Only 的源域评估准确率达到 0.8767，而目标域准确率仅有 0.5203。该现象说明“源域分类做得好”并不等价于“目标域迁移做得好”，领域自适应方法的核心价值在于缩小这一源-目标性能落差。

与此同时，基线实验也显示某些方法虽然加强了域对齐，却不一定能在所有场景上带来正收益。例如 mode6$\rightarrow$mode3 场景中，部分方法在目标域上反而低于 Source Only。这意味着在工业多类别时序任务中，单纯进行全局对齐可能会压缩原本较好的类别结构，进而造成负迁移。

\section{讨论与方法启示}
\subsection{当前结果揭示的主要问题}
结合当前实验结果，可以总结出以下几个值得关注的问题。

\begin{enumerate}
\item \textbf{域偏移影响依然显著。} Source Only 在源域上通常能够获得较高准确率，但在目标域上的性能下降十分明显，说明跨工况分布偏移仍然是该任务的核心困难。
\item \textbf{全局对齐有效但不充分。} DAN 和 DANN 证明，只要能够有效缩小源域与目标域整体分布差异，目标域性能就能获得显著提升；然而，仅有全局对齐并不能保证跨域同类样本一定聚合。
\item \textbf{场景敏感性明显存在。} 同一方法在不同迁移方向上可能表现迥异。特别是当 Source Only 已较强时，过强的对齐反而可能导致负迁移。
\item \textbf{条件结构利用仍不稳定。} CDAN 在理论上能够利用类别信息，但在当前场景中的平均表现未能持续超越 DANN，说明伪条件信息与模型选择策略之间仍存在耦合问题。
\end{enumerate}

\subsection{对第五章方法设计的启示}
上述现象表明，当前问题的关键并不只是“是否进行对齐”，而是“如何在保持类别判别能力的前提下进行更稳定的对齐”。现有全局边缘分布对齐方法能够减小整体域差异，但无法显式保证跨域同类样本靠近、异类样本保持分离；而条件对齐方法虽然引入了类别信息，却仍容易受到伪标签噪声和训练不稳定性的影响。因此，改进方法至少需要同时兼顾以下三点：

\begin{enumerate}
\item 保留当前实验中已经验证有效的全局域对齐机制，避免完全脱离现有强基线；
\item 引入更直接的类别结构约束，以减少跨域类别混叠并增强同类样本聚合；
\item 通过可靠性筛选和课程式权重调度，控制目标域伪标签噪声对训练的负面影响。
\end{enumerate}

基于这一思路，下一章将提出可靠性门控的类条件时序自适应方法 RCTA，并给出其损失设计、训练策略和当前验证结果。

\section{本章小结}
本章在统一 FCN 骨干网络和统一实验协议下，对 Source Only、CORAL、DAN、DANN、CDAN 与 DeepJDOT 等基线方法进行了比较。当前已完成的四个代表性单源场景结果表明，领域自适应方法整体上显著优于 Source Only，其中 DANN 的平均目标域表现最强，DAN 则展现出较好的跨场景稳定性。与此同时，基线实验也揭示出全局对齐不足、场景敏感和条件结构利用不稳定等问题。基于这些结果，下一章将进一步给出 RCTA 的方法设计与阶段性验证。
