\chapter{绪论}

\section{研究背景与意义}
流程工业广泛存在于化工、能源、冶金、制药、环保和材料制造等关键行业，其生产过程通常具有连续运行、物料与能量强耦合、动态特性显著、控制回路复杂以及安全风险高等特点。一旦关键设备或工艺环节发生异常，不仅会导致产品质量波动、能耗上升和非计划停机，还可能进一步诱发连锁故障甚至安全事故。因此，如何及时、准确地发现并识别工业过程中的故障，一直是过程监测、运行优化和安全保障中的重要研究问题~\cite{qin2012survey}。

随着分布式控制系统、工业互联网和边缘采集装置的普及，工业现场能够持续积累大规模、多变量、高频率的过程数据。与传统依赖专家经验、机理模型或阈值规则的方法相比，数据驱动故障诊断方法能够在较弱先验条件下从历史运行数据中自动提取判别特征，因此已经成为工业过程监测的重要技术路线之一~\cite{qin2012survey}。尤其是在深度学习方法进入故障诊断领域之后，卷积神经网络、循环神经网络以及各类时序表征模型进一步提升了模型对复杂动态模式的建模能力。

然而，工业故障诊断中的一个核心困难在于训练分布与部署分布往往并不一致。设备老化、原料性质变化、环境扰动、控制参数重整定、负载切换以及生产阶段变更等因素，都会造成不同时间段、不同工况乃至不同装置上的过程数据分布发生偏移。设训练阶段的数据分布为 $P_s(x,y)$，部署阶段的数据分布为 $P_t(x,y)$，在很多真实场景下有
\begin{equation}
P_s(x,y)\neq P_t(x,y).
\end{equation}
当模型仅利用源域标注数据训练时，即使其在源域上取得较高准确率，也可能在目标域上出现显著性能下降。这一问题在流程工业中尤为突出，因为目标工况下的故障样本往往稀缺、代价高昂且带有较强风险属性，重新采集并人工标注大量目标域故障数据通常并不可行~\cite{pan2009survey}。

领域自适应作为迁移学习的重要分支，正是为缓解上述分布偏移问题而提出。其核心思想是在源域有标签、目标域少标签甚至无标签的条件下，通过特征对齐、对抗学习、类别约束或最优传输等方式提升模型在目标域上的泛化能力。对于流程工业而言，如果能够在不依赖目标域大量标注样本的条件下稳定提升跨工况故障识别性能，将对提升模型的工程可用性、降低部署成本和缩短模型迁移周期具有实际意义。

Tennessee Eastman Process（TEP）是过程系统工程领域广泛使用的经典工业过程仿真平台，由 Downs 和 Vogel 提出~\cite{downs1993plant}。围绕 TEP 构造的领域自适应数据集将不同运行模式组织为不同领域，使得“跨工况故障诊断”能够被清晰表述为无监督领域自适应任务。该数据集既保留了工业多变量时序数据的动态特征，又提供了较规范的任务边界，因此非常适合用于构建统一实验基准、比较典型方法并验证新方法的有效性。

\section{研究问题与核心挑战}
本文关注的核心问题是：在目标域缺少故障标签的条件下，如何利用源域标注样本与目标域无标注样本，训练一个能够跨工况迁移的故障诊断模型。围绕这一问题，本文将研究场景限定为单源到单目标的无监督领域自适应诊断任务。源域与目标域分别对应不同的 TEP 操作模式，模型输入为多变量过程时序片段，输出为对应的正常/故障类别预测。

与一般图像分类或标准迁移学习问题相比，本文任务至少面临以下几个方面的挑战。

\subsection{跨工况分布偏移显著}
不同运行模式下的负载水平、控制策略和动态响应均可能不同，因此同一故障类别在不同模式下表现出的时序轨迹并不完全一致。也就是说，目标域不仅存在边缘分布变化，
\begin{equation}
P_s(x)\neq P_t(x),
\end{equation}
还可能伴随条件分布差异，
\begin{equation}
P_s(y|x)\neq P_t(y|x).
\end{equation}
这使得简单依赖源域监督训练的分类器难以保证跨域泛化效果。

\subsection{目标域无标签，监督信号间接}
在无监督领域自适应设定中，训练阶段只能使用源域标签，目标域样本虽然可见，但其标签不可用。因此训练过程无法直接最小化目标域分类误差，只能依赖无标签代理指标、伪标签、域对齐或结构约束来间接逼近目标风险。如何在不过度引入噪声监督的前提下充分利用目标域样本，是方法设计的关键。

\subsection{工业时序数据类别结构复杂}
TEP 领域自适应数据集包含 29 个类别，既包括正常工况也包括多种故障类型。与若干低类别数迁移任务相比，29 类流程工业时序数据在跨域条件下更容易出现类别混叠和决策边界不稳定的问题。仅依赖全局域对齐，可能会把跨域异类样本压缩到相近区域，反而带来负迁移。

\subsection{实验比较需要统一且可复现}
领域自适应方法往往对骨干网络、数据预处理、选择指标和超参数高度敏感。如果没有统一的实验协议，不同方法之间的差异很容易被数据划分和训练细节掩盖。因此，在进行方法比较之前，必须先建立统一的数据组织方式、训练脚手架和结果管理流程。

\section{研究内容与技术路线}

\subsection{研究内容}
围绕上述问题，本文的工作主要包括以下三个方面。

\begin{enumerate}
\item 面向 TEP 多工况迁移诊断任务，建立统一的数据读取、预处理、训练、评估与结果汇总流程，在统一协议下组织单源到单目标领域自适应实验；
\item 在共享 FCN 骨干网络和一致训练设定下，系统比较 Source Only、CORAL、DAN、DANN、CDAN 与 DeepJDOT 六种典型方法在四个关键单源场景上的表现，分析不同对齐机制的收益与局限；
\item 针对基线实验揭示出的类别混叠、伪标签噪声和场景敏感性问题，提出可靠性门控的类条件时序自适应方法 RCTA，并围绕其结构设计与实验结果开展论证。
\end{enumerate}

\subsection{技术路线}
本文的技术路线可以概括为以下四步。

\begin{enumerate}
\item 首先，对 TEP 领域自适应数据集进行任务化建模，明确域定义、样本组织、归一化方式、输入表示和统一评价指标；
\item 其次，以一维全卷积网络作为共享时序特征提取骨干，在一致实验协议下复现多种典型领域自适应基线方法；
\item 再次，基于基线结果总结流程工业跨工况迁移中的主要矛盾，并据此提出 RCTA 的教师--学生协同、可靠性门控、双原型记忆与课程调度机制；
\item 最后，围绕四个关键单源场景，对基线方法与 RCTA 进行定量与定性分析，形成从实验基准构建到方法改进的完整研究链路。
\end{enumerate}

\section{本文的主要工作与创新点}
本文的主要工作与创新点可概括为以下四点。

\begin{enumerate}
\item 构建了面向 TEP 跨工况诊断的统一单源无监督领域自适应实验框架。该框架统一了数据组织方式、输入表示、归一化策略、骨干网络、模型选择规则与结构化结果产物，为后续公平比较奠定了基础。
\item 在统一协议下完成了六种代表性基线方法的系统比较，并围绕四个关键单源场景给出了可复核的结果表、热力图和定性分析，明确揭示了全局对齐有效但不充分、迁移方向难度不对称以及负迁移可能出现等现象。
\item 提出了可靠性门控的类条件时序自适应方法 RCTA。该方法以强基线对齐分支为底座，引入 EMA 教师网络、弱强双视图时序增强、多因素可靠性评分、按类门控伪标签采纳、双原型记忆与课程化损失调度，以更稳定地利用目标域无标签样本。
\item 在四个关键单源场景上，RCTA 均取得最优目标域准确率。实验结果表明，RCTA 的平均目标域准确率达到 0.8590，相比 Source Only 平均提升 0.1884，相比逐场景最强基线平均进一步提升 0.0186，验证了所提方法的有效性。
\end{enumerate}

\section{论文结构安排}
全文共分为六章，各章内容安排如下。

第一章为绪论，介绍研究背景、任务挑战、研究内容以及本文的技术路线。第二章为相关基础与文献综述，梳理流程工业故障诊断、领域自适应方法以及 TEP 跨工况迁移诊断的研究脉络。第三章为问题定义、数据集与实验设定，系统说明任务形式、数据组织、输入表示、统一骨干网络与模型选择策略。第四章为基线方法与实验结果分析，重点比较六种典型方法在四个关键单源场景上的表现，并总结主要实验现象。第五章为可靠性门控的类条件时序自适应方法设计与实验验证，重点阐述 RCTA 的架构创新、优化目标以及定量定性结果。第六章为总结与展望，对全文工作进行归纳，并讨论进一步向更复杂迁移场景与工程应用扩展的方向。
