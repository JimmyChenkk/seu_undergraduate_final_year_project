# 项目升级与更强创新点草案

目标：
- 当“RCTA 作为机制组合改进”已经足够支撑本科论文时，这份文档用于思考下一步如何把项目升级成更有辨识度的研究主题。
- 所有方案都尽量基于当前代码结构可延展的方向，而不是完全推翻重来。

## 方向一：多源可靠性加权 RCTA

### 核心想法
把当前的 reliability-gated 机制从“目标样本可靠性”扩展到“源域可靠性 + 源域关系建模”。

### 为什么更强
- 当前 benchmark 已有 3 个多源场景，天然具备实验舞台。
- 现有多源设置更像“把多个源域拼起来一起训”，但没有回答“哪个源域更值得迁移”。
- 如果能给每个源域学习一个动态权重，就比单纯多源融合更像一个明确创新点。

### 可以怎么做
- 基于源域原型与目标域原型之间的距离，为每个源域估计场景级权重。
- 基于每个源域在目标域上的无标签代理表现，学习 batch-level source weight。
- 对抗分支、伪标签分支和原型分支分别使用不同的 source weight，而不是统一加权。

### 最小实验闭环
- 在当前 3 个多源场景上对比“简单合并多源”和“可靠性加权多源”。
- 增加一张源域权重可视化图，说明不同场景下模型偏好哪个源域。

## 方向二：流程先验引导的结构化 RCTA

### 核心想法
把当前 purely feature-space 的 prototype 结构，升级为“变量关系图 + 原型结构”联合建模。

### 为什么更强
- 这是最能体现“工业过程特色”的方向。
- 能把论文从“通用迁移学习方法套到 TEP 上”升级为“结合流程工业先验的任务特化方法”。

### 可以怎么做
- 按工艺变量分组构建 channel groups，在编码器中加入组内/组间约束。
- 基于已知控制回路或变量相关性构建 graph regularization。
- 对 prototype similarity 的计算加入变量组注意力，而不是只用全局 cosine similarity。

### 最小实验闭环
- 先做一个 lightweight 版本：变量分组加权原型相似度。
- 证明其在易混类别上的混淆矩阵更清晰，而不是只看平均准确率。

## 方向三：在线/测试时自适应版 RCTA

### 核心想法
把当前离线 UDA 扩展为 test-time adaptation 或 online adaptation。

### 为什么更强
- 更贴近真实工业部署。
- “可靠性门控”天然适合做在线样本筛选。
- “原型记忆”天然适合做流式更新。

### 可以怎么做
- 推理阶段使用 gate 选择高可靠样本更新目标原型。
- 引入 drift detector，当目标域分布明显变化时提升更新频率。
- 保留教师网络和原型，冻结或半冻结主干网络，只做轻量更新。

### 风险
- 实验协议会从标准 UDA 变成在线适应，需要重新定义公平比较基线。
- 本科论文阶段如果时间紧，这条线更适合作为展望或后续工作。

## 方向四：开放集/未知故障拒识

### 核心想法
把 closed-set UDA 扩展为 open-set fault diagnosis，让模型在遇到未知故障时能够拒识而不是强行分到已知类别。

### 为什么更强
- 工业场景里很真实。
- 当前 reliability gate 和 prototype distance 都能作为 unknown score 的基础。

### 可以怎么做
- 用最大 softmax probability、entropy、prototype distance 组合出 unknown score。
- 在目标域引入“已知类适配 + 未知类拒识”双目标。
- 评估指标从单纯 accuracy 扩展为已知类准确率、未知类拒识率、H-score。

### 风险
- 数据集需要支持未知类划分，可能需要额外构造协议。

## 我最推荐的升级顺序

1. 先做“多源可靠性加权 RCTA”。
2. 再做“流程先验引导的结构化 RCTA”。
3. 如果还有时间和资源，再考虑在线适应或开放集。

## 为什么这样排

- 多源可靠性加权最贴合当前 benchmark，改动最小，最容易产出能讲清楚的新增实验。
- 流程先验引导最能增强项目辨识度，适合下一步把论文从“方法组合”提升为“工业任务特化创新”。
- 在线适应和开放集都很强，但会明显扩大任务边界，更像后续研究方向。
