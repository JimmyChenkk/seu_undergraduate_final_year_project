# RCTA 论文收束策略备忘录

## 结论

当前项目应进入论文收束阶段：冻结现有 RCTA 主方法，不再继续做大规模调参或主算法升级。下一步重点不是追求单场景更高准确率，而是补齐论文证据链，包括主 benchmark 回填、代表性模块消融、定性可视化和边界讨论。

基于 `20260423_155127_benchmark_88_2clusters_11scenes_8methods_fixedfold` 这一轮结果，RCTA 已经足够支撑本科毕设主线：

- RCTA 在 11 个 fixed-fold 场景上的平均目标域准确率为 0.8213，是非监督参考方法中最高。
- Source Only 平均为 0.5898，Target Only 平均为 0.9124，RCTA 已经显著缩小跨工况迁移差距。
- 排除 Target Only 后，RCTA 在 11 个场景中取得 8 个第一。
- RCTA 平均排名为 1.91，优于 CDAN、DAN、DANN、DeepJDOT、CORAL 和 Source Only。
- RCTA 仍有边界场景，尤其是 `mode3->mode6`、`mode1->mode2`、`mode2->mode1`，不适合表述为所有场景全面最优。

因此，论文中应采用稳健表述：RCTA 在统一 fixed-fold benchmark 下取得最优平均性能和最佳平均排名，说明可靠性门控、时序一致性和类条件原型约束能够整体提升跨工况迁移稳定性；但在部分特定迁移方向上，传统对抗或统计对齐方法仍可能更优，说明跨工况故障诊断存在场景依赖性。

## 消融实验原则

机制组合类 UDA 方法不要求每个模块在每个迁移场景都单调提升。Mean Teacher、一致性正则、置信度筛选、prototype memory 和 curriculum training 本来就是强交互组件，拆开后局部不稳定是可以解释的。

消融实验的目标应是证明完整组合在以下方面有总体贡献：

- 平均性能更好；
- 困难场景表现更稳；
- 训练过程更稳定；
- 伪标签质量更可控；
- 类别结构更清晰。

论文中不要写“每个模块均能在所有迁移任务上稳定提升性能”。更稳妥的写法是：

> 消融结果表明，各模块对完整框架具有互补贡献。虽然个别迁移任务中某一模块的单独增益不明显，但从平均识别率、困难迁移任务表现和训练稳定性来看，完整模型取得了更优的综合效果。

## 推荐最小消融闭环

不建议做完整 `88-run x 多变体` 消融。当前最合适的是小规模代表性场景消融。

推荐范围：

- 优先方案：4 个代表性场景 x 5 个变体，共 20 个 run。
- 更省时间方案：3 个代表性场景 x 5 个变体，共 15 个 run。

推荐场景：

- `mode2->mode5`：RCTA 优势明显的代表性困难场景。
- `mode5->mode2`：RCTA 优势明显，且与上一场景方向相反。
- `mode3->mode6`：RCTA 边界场景，CDAN 明显更强，可用于诚实讨论局限。
- `mode1->mode2`：RCTA 有提升但不是最优，可用于观察模块在竞争场景中的贡献。

如果只保留 3 个场景，可去掉 `mode5->mode2`，保留 `mode2->mode5`、`mode3->mode6`、`mode1->mode2`。

推荐变体：

| 变体 | 含义 | 目的 |
| --- | --- | --- |
| Base DA | 只保留基础 DANN/MCC 对齐 | 作为适配底座 |
| Base + MT | 加 EMA teacher / 一致性入口 | 验证教师预测稳定性 |
| Base + MT + Gate | 加可靠性门控伪标签 | 验证目标样本筛选 |
| Base + MT + Gate + Proto | 加双原型记忆 | 验证类条件结构约束 |
| Full RCTA | 加课程训练与置信保护选择 | 验证完整组合 |

如果工程上更容易，也可以采用 w/o 形式：

| 变体 | 含义 |
| --- | --- |
| Full RCTA | 完整方法 |
| RCTA w/o MT | 去掉 teacher-student 一致性 |
| RCTA w/o Gate | 去掉可靠性门控或退化为固定阈值 |
| RCTA w/o Proto | 去掉双原型记忆 |
| RCTA w/o Curriculum | 去掉课程式调度与置信保护选择 |

本科论文更推荐递进式组合，因为它与第三章到第六章的章节结构更一致。

## 消融指标

消融表不应只放目标域 Accuracy。建议至少保留：

- Target accuracy；
- Target balanced accuracy；
- 平均值；
- selected pseudo-label ratio 或 gate accept ratio；
- pseudo-label mean reliability；
- t-SNE 或 UMAP 可视化；
- 混淆矩阵；
- 训练曲线或 selected epoch。

如果能计算伪标签准确率，可以作为分析指标出现，但需要明确它只用于训练后分析，不参与训练或模型选择。

## 论文表述模板

### 消融实验设计

> 为验证 RCTA 中各组成模块的有效性，本文在若干具有代表性的迁移场景上进行了模块消融实验。具体而言，在基础域对齐模型之上，逐步引入 teacher-student 一致性约束、伪标签可靠性门控、类别原型记忆模块以及课程式训练策略，并比较不同变体在目标域测试集上的识别性能。考虑到完整全场景消融的计算成本较高，本文选取包含不同迁移难度的代表性任务进行分析，同时结合主实验中的全部迁移结果评估方法的整体有效性。

### 模块不单调提升时

> 需要指出的是，部分模块在个别迁移任务中的单独增益并不稳定。这一现象与无监督领域自适应任务本身的特点有关：目标域标签不可见，伪标签质量受域偏移、类别混淆和模型置信度校准程度影响较大。因此，本文并不将单个模块表述为在所有场景下均能带来单调提升，而是从平均性能、训练稳定性和困难任务表现三个角度评价其对完整框架的贡献。

### 完整模型优势解释

> 实验结果表明，RCTA 的性能提升主要来自多个可靠性约束的协同作用。Mean Teacher 机制有助于提供更平滑的目标域预测，置信度筛选减少低质量伪标签的干扰，原型记忆模块强化类别结构约束，课程式训练则控制目标域伪标签逐步参与训练的节奏。各模块之间存在互补关系，因此完整模型相比单独模块变体表现出更好的综合性能。

### 最稳妥结论

> 综上，消融实验说明 RCTA 中各模块并非在所有迁移任务中都表现为独立、单调的性能增益，但它们在完整框架中能够共同提高目标域伪标签的可靠性和训练过程的稳定性，从而在多数代表性场景及平均结果上取得更优表现。

## 多源可靠性加权 RCTA 的定位

多源可靠性加权 RCTA 是非常自然的后续方向，但不建议现在作为主方法实现。原因是它会新增方法、公式、实验、图件和文献工作，容易把项目从收尾重新拉回开题。

当前论文中建议将其放入第八章展望：

> 现有 RCTA 主要从目标域样本可靠性角度控制伪标签利用，原型记忆主要用于稳定类别结构。然而，在实际工业过程故障诊断中，历史数据往往来自多个工况或多个运行模式，不同源域与目标域之间的相关性并不一致。若简单合并多个源域，可能引入低相关源域造成负迁移。因此，后续工作可在 RCTA 基础上引入源域可靠性加权机制，根据源域与目标域的分布距离、类别原型相似度以及预测一致性动态估计各源域贡献，从而实现更稳健的多源跨工况故障诊断。

这一方向的核心逻辑是从“目标样本可靠性”扩展到“源域、源类或源样本可靠性”。可考虑的设计包括：

- 基于 source-target prototype distance 估计源域权重；
- 基于 source classifier confidence 或 teacher-student agreement 估计源域贡献；
- 对分类损失、对齐损失和原型损失分别使用不同的源域权重；
- 在多源场景中可视化源域权重，解释哪些历史工况更接近目标工况。

但在当前论文中只作为后续工作，不作为已经完成的实验结论。
