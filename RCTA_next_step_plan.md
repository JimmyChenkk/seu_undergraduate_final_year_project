# RCTA 下一阶段计划

## 1. 当前状态

当前项目已经完成了 TEP 跨域诊断 benchmark 的基础搭建，并且 RCTA 主体框架已经具备以下核心组件：

- teacher / EMA 机制
- pseudo-label 训练
- reliability gating
- prototype 约束
- consistency 正则
- 多种基础域适应对齐策略

因此，下一阶段的工作重点不再是继续堆叠模块，而是围绕**目标域伪标签质量、跨域鲁棒性与训练稳定性**做系统性优化。

## 2. 当前判断

本阶段对 RCTA 的判断是：

- **方向没有问题**，已经具备继续迭代的基础。
- **仍有提升空间**，尤其是伪标签质量、不确定性建模、课程式训练和原型结构约束。
- **暂时不急于立即升级实现**，可以先完成路线收紧、文献补充和实验设计整理。

换句话说，当前更适合先把“后续怎么改”想清楚，而不是马上进入大规模代码改造。

## 3. 下一阶段主线

下一阶段建议围绕下面一条连续主线展开：

> **伪标签校准 → 可靠性递进训练 → 原型结构约束**

对应关系如下：

1. **伪标签校准**
   - 从硬性的伪标签筛选，升级为样本级可信度评估。
   - 目标是让高可信样本更充分参与监督，低可信样本以更温和的方式参与训练。

2. **可靠性递进训练**
   - 引入课程式伪标签学习。
   - 按照样本置信度由高到低逐步吸收目标域信息，降低早期错误伪标签的累积放大风险。

3. **原型结构约束**
   - 结合原型约束与对比学习。
   - 增强类内紧凑性与类间可分性，提升跨域特征对齐稳定性。

## 4. 优先级建议

如果后面开始升级 RCTA，建议按以下顺序推进：

### 第一优先级

- 置信度校准
- 不确定性感知伪标签
- 样本级可靠性评估

原因：

- 与现有框架耦合最小
- 最容易做 ablation
- 直接针对伪标签噪声问题

### 第二优先级

- 课程式伪标签训练
- 分阶段引入 target 样本
- 高置信到低置信的递进式监督

原因：

- 与现有 teacher / pseudo-label 结构天然兼容
- 适合作为完整方法的训练策略部分

### 第三优先级

- 原型约束增强
- prototype-aware contrastive learning
- 类间结构约束

原因：

- 有助于提升类结构稳定性
- 但实现和调参复杂度稍高

## 5. 近期不立即做的内容

近期如果时间不允许，可以先不做以下方向：

- 大规模重构 backbone
- 重型 self-attention 主干替换
- Mixture of Experts
- 因果表示学习主线
- 全套 Bayesian / ensemble 方案
- 复杂图神经网络主干

这些方向不是不能做，而是短期内不一定最划算，尤其在当前阶段更适合作为备选或未来工作。

## 6. 近期工作安排建议

### A. 先整理研究路线

目标：把 RCTA 的后续改进思路固定下来。

建议产出：

- 一段正式的“下一阶段提升方向”表述
- 一个可用于和网页 chatbot 继续讨论的短版
- 一份模块 / loss / ablation 对照草案

### B. 先不急着改代码

如果当前还在做 benchmark 稳定化或其他实验，可以先保持现有实现不动，等到以下条件满足后再动手：

- 现有 benchmark 结果基本稳定
- 关键方法对比数据已经有初步结论
- 后续改进方向已经足够明确

### C. 等到合适时再做最小改动版本

如果开始实现，建议先做最小闭环：

1. 先加置信度校准或 uncertainty score
2. 再把 hard gate 改成软门控
3. 再加入 curriculum 训练
4. 最后再考虑 prototype / contrastive 增强

## 7. 实验验证原则

未来如果启动 RCTA 升级实验，建议遵循以下原则：

- 先做小规模验证，再做批量实验
- 每次只引入一个主要变化，便于消融
- 优先比较稳定性，而不是只看单次最好值
- 关注 target accuracy、source_eval、混淆矩阵和 t-SNE
- 尽量保留相同训练协议，避免把结论混淆成调参效果

## 8. 当前结论

当前可以先把 RCTA 的下一阶段方向定为：

- **不急于马上改代码**
- **先收紧研究主线**
- **先明确模块和实验设计**
- **等 benchmark 主流程更稳定后，再进入最小改动式升级**

这会比立刻大改更稳，也更符合当前项目节奏。
